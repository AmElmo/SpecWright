# Configure review preferences

**Issue ID**: ENG-005  
**Status**: pending  
**Category**: fullstack  
**Complexity Score**: 4/10  
**Estimated Hours**: 3h  
**Complexity Reasoning**: Standard CRUD settings page with toggle UI - familiar pattern but needs database schema.

## Description

Let users customize which categories of issues to check. Preferences are saved to their profile and applied to all future reviews.

## Technical Details

- Settings page with toggle switches for each category
- GET/PUT `/api/users/review-preferences` endpoints
- Database table `review_preferences` with user_id foreign key
- Backend filters AI prompts based on enabled categories (saves tokens)

## Dependencies

- User authentication system (to associate preferences with user)

## Acceptance Criteria

- [ ] Settings page shows four category toggles
- [ ] Toggles reflect current saved preferences
- [ ] Changes save automatically (or with Save button)
- [ ] Toast confirms "Preferences saved"
- [ ] New reviews respect the preferences
- [ ] Disabled categories not sent to AI (saves tokens)
- [ ] Default: Security, Bugs, Performance ON; Style OFF

## Test Strategy
